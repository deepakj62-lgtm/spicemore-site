Direct Edit — iPhone / iPad Setup
Edwin — this gives you full edit + push access to spicemore.com from your iPad or iPhone. No laptop. No relay through Deepak. Your edits go live in ~60 seconds.

You'll be using Claude Code on your iPad/iPhone. It's a version of Claude that can directly edit files and push them to the website. Setup takes ~10 minutes once. After that, every change is just a sentence.

Heads up — full access means full responsibility. Deepak is giving you direct-push to production. A bad commit goes live immediately. Read the guardrails section at the bottom before you start. If you ever want to test something risky, ask Deepak first.
First-time setup — 10 minutes, do once
1Get Claude Pro (if you don't have it)
  • $20/month. Includes Claude Code access on iPad/iPhone.
  • Open the App Store → search Claude → install → sign in with your email.
  • In the Claude app: tap your profile (bottom-right) → Settings → Subscription → upgrade to Pro if you haven't.
2Create a GitHub account (if you don't have one)
  • Open Safari → github.com/signup
  • Use any email. Pick a username — write it down, you'll send it to Deepak so he can add you to the spicemore-site repo.
  • Verify your email when prompted.
3Send Deepak your GitHub username
  • WhatsApp him: "My GitHub username is <your-username>"
  • He'll add you as a collaborator on github.com/deepakj62-lgtm/spicemore-site.
  • You'll get an email invite — open it on your iPad and tap Accept invitation.
4Generate a GitHub Personal Access Token
This is a long string that lets Claude Code push to the repo on your behalf.
  1. In Safari: github.com/settings/tokens?type=beta
  2. Tap Generate new token → Fine-grained personal access token.
  3. Name: spicemore-claude-code
  4. Expiration: 1 year (or longer — your call).
  5. Repository access: Only select repositories → pick deepakj62-lgtm/spicemore-site.
  6. Permissions → Repository permissions:
    • Contents: Read and write
    • Pull requests: Read and write
    • Metadata: Read-only (auto-selected)
  7. Tap Generate token.
  8. Copy the token (long string starting with github_pat_...) — save it somewhere safe right now. Apple Notes works. You won't be able to see it again.
Treat this token like a password. Don't share it on WhatsApp, don't paste it in random chats, don't email it. If you think it leaked, go back to that GitHub page and revoke it — generate a new one.
5Open Claude Code on your iPad
  • In the Claude app: tap the + New chat button.
  • In the chat type-bar, look for the Code mode toggle (or a "Claude Code" option in the chat-type picker). Tap it. (If you can't find it, just open Safari and go to claude.ai/code — same thing in browser.)
  • You'll see a coding-style chat with a sandbox terminal underneath. That's your private cloud Linux machine — anything you do here stays in your Claude account.
6Tell Claude Code about your GitHub token
One-time. After this, Claude Code remembers and you never type the token again.

In your first Claude Code session, type this (paste your actual token where it says github_pat_xxx):

Set up my GitHub credentials. My username is <your-github-username> and my Personal Access Token is github_pat_xxx. Configure git globally so I can push to deepakj62-lgtm/spicemore-site without re-entering credentials each time.

Claude Code will run the right git config commands and store the token in its session. Test that it worked:

Clone deepakj62-lgtm/spicemore-site into ~/work and confirm you can see the file sepl/staff/intake.html

If Claude Code reports the file exists, you're set up. ✅

You're done with one-time setup. From here on, every change is one or two sentences in Claude Code.
Everyday flow — making a change
Once setup is done, here's how every edit works. ~2 minutes per change.
1Open Claude Code, describe what you want

Plain English. Examples:

Tweak text or copy 
In spicemore-site, change the warning text on sepl/staff/intake.html from "Only click after the client has committed the intake" to "Only click after the client has signed the intake form". Commit with a clear message and push.
Change colours / styling 
On sepl/staff/intake.html, change the warning banner background from yellow to red. Use the existing CSS variables — don't add hex colours. Commit and push.
Add a new field 
On sepl/staff/intake.html in the Client section, add a new optional field called "Aadhaar number" right below the Phone field. Match the styling of the other inputs. Commit and push.
Reorder sections 
On sepl/staff/intake.html, swap the order of the "Sample test" block and the "Depot" dropdown so Depot comes first. Commit and push.
2Claude Code does it
  • Reads the file, makes the change, commits with a clear message, pushes to main.
  • Vercel automatically deploys to spicemore.com in ~30-60 seconds.
  • Claude Code will tell you when it's done.
3Verify on the live site
  • Open spicemore.com/sepl/staff/intake on your iPad.
  • Pull-to-refresh. Check your change is live.
  • If it's not what you wanted, just tell Claude Code: "undo that change" or "revert the last commit" — it'll do it.
Bigger changes: If you want to test a risky change without going live first, ask Claude Code to: "create a branch called edwin/test-X, do the change there instead of main, push, and give me the preview URL." Vercel auto-builds a preview URL for any branch — you can test there before merging.

Guardrails — please don't do these
NEVER tell Claude Code to:
  • Delete the repo or do git push --force on main
  • Change anything in api/ (the backend code) — those are running calculations and money flow. Anything in api/ needs Deepak to review.
  • Change the agreement PDF text in api/sepl/agreement-pdf.js — that's legal text Edwin signed off on Apr 15.
  • Change LTV thresholds or holding rate in api/sepl/_settings.js — those are the economic terms.
  • Push secrets / tokens / passwords into the repo. If Claude Code suggests committing a .env file, say no.
If Claude Code asks "should I do X" and you're not sure — answer NO. WhatsApp Deepak with what it asked. Better to wait 5 minutes than break production.
Safe to do anytime:
  • Edit anything in sepl/staff/*.html (the staff-facing pages — your main editing surface)
  • Edit anything in sepl/consignor/*.html (client-facing pages)
  • Edit sepl/index.html (SEPL portal landing page)
  • Change CSS in css/style.css
  • Update text, copy, layout, fields, validation messages, error messages, tooltips, button labels
  • Add new pages under sepl/
Troubleshooting
"Claude Code says it can't push — permission denied"
  • Token expired or invalid. Generate a new one (step 4 above) and tell Claude Code: "Update my GitHub token to github_pat_NEW_TOKEN_HERE".
  • Or you haven't accepted the collaborator invite yet — check email.
"My change isn't showing on spicemore.com"
  • Wait 60 seconds. Vercel takes a moment to build + deploy.
  • Hard-refresh: pull-to-refresh in Safari, or close and reopen the tab.
  • Check it's actually deployed: ask Claude Code: "What's the current deployment status of spicemore.com?" — it can check Vercel for you.
"Claude Code edited the wrong file"
  • Tell it: "That was the wrong file — revert that change and try again on FILE_NAME instead."
  • If it already pushed, ask it to revert the commit and re-do.
"I broke something and don't know how to fix it"
  • WhatsApp Deepak. Don't keep poking at it. He can revert from his side in 30 seconds.
  • Send him: the page that's broken, what you tried last, and what error you see.
"Claude Code is being weird / slow / not understanding"
  • Start a new chat. Sometimes the session gets confused. Each new chat is fresh.
  • Be more specific. Instead of "fix the warning" say "in sepl/staff/intake.html, change the text inside the div with class commit-warning to: ..."
